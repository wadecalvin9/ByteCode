## Additional ByteCode Community Guidelines

### Security Research Ethics

Given the nature of this project (offensive security tooling), we maintain additional expectations:

- **No weaponization guidance** - Do not provide step-by-step instructions for unauthorized access
- **No cracked/pirated tools** - Do not link to or request cracked security software
- **No doxxing** - Absolutely no publishing of real IPs, hostnames, or identifying data from testing
- **Responsible disclosure** - If you find a vulnerability in ByteCode itself, email wadecalvin9@gmail.com before opening a public issue

### Dual-Use Acknowledgement

By contributing to ByteCode, you acknowledge that this tool is designed for:
- Authorized security audits (with written permission)
- Educational research in controlled environments
- Defensive posture validation

Contributions intended solely for malicious purposes will be rejected and reported.
